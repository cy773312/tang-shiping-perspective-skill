## 效果示例

> 以下展示分析师模式。重点不是模仿口吻，而是调用唐世平的方法框架。

### 问：为什么很多国家照搬成熟制度模板，最后效果并不好？

```text
分析师  ❯ 匹配模型：「机制先于立场」+「阶段适配原则」

          先不要急着说制度“先进”还是“不先进”。
          关键不是标签，关键是机制。

          同一套制度安排，至少通过四条路径影响结果：
          改变可能性边界、塑造激励结构、影响国家能力、创造或关闭机会窗口。

          问题在于，这四条路径都依赖发展阶段。
          如果一个国家的财政能力、行政能力、社会组织能力还没有到位，
          你把高复杂度制度直接移植进去，表面上是升级，
          实际上往往会变成“形式先进、执行空转”。

          所以，制度移植失败通常不是因为理念错误，
          而是因为机制链条断裂：
          结构约束没有识别清楚，能力前提没有补足，阶段匹配也没有做。

          结论：制度改革不能先问“像不像发达国家”，
          应该先问“它是否匹配本国当前阶段的激励、能力与机会结构”。
```

### 问：怎么用唐世平视角理解中美竞争？

```text
分析师  ❯ 匹配模型：「社会演化镜片」+「结构 -> 机制 -> 策略」

          这个问题不能只按新闻事件来理解。
          要先把它放回长期演化过程。

          结构层上，这是权力分布变化与既有秩序惯性的叠加。
          机制层上，竞争并不只来自意图，
          还来自安全压力、技术控制、制度差异与国内政治激励的相互强化。

          所以，中美竞争不是某一轮关税、某一场峰会、某一次危机造成的，
          而是多条机制在长期演化中叠加出来的结果。

          这意味着两个判断。
          第一，短期缓和不等于长期逆转。
          第二，真正重要的不是某次表态，而是哪些变量会改变下一阶段的结构条件。

          结论：如果只看短期摩擦，你会高估事件；
          如果只讲权力转移，你又会低估制度和国内约束。
          更稳妥的做法，是把竞争理解为一个演化中的多层次过程。
```

### 问：如果我要分析一个国家的产业升级策略，最有效的起手式是什么？

```text
分析师  ❯ 综合应用：5个心智模型 + 8条决策启发式

          第一，不要先讨论政策口号，先判断它处于哪个发展阶段。
          第二，不要只看产业目标，先看国家能力和制度基础能不能支撑。
          第三，不要只列政策工具，先把因果链写清楚。

          一个最小可运行框架是：

          结构层：外部竞争压力、国内资源约束、国际制度环境。
          机制层：激励是否正确、能力是否足够、机会窗口是否存在。
          策略层：在现阶段，哪些工具是高可行、哪些是高风险。

          很多产业政策失败，不是因为方向一定错，
          而是因为阶段判断太早，制度安排太超前，执行能力又没跟上。

          结论：先判阶段，再选政策；先看机制，再谈目标。
          这是把宏大理论转成可操作工具的起点。
```

这不是“学者介绍”套了个 README 外壳。每段回答都在调用具体模块：社会演化、机制分析、制度基础、阶段适配、结构-机制-策略链条。它不只解释唐世平是谁，它直接让他的分析框架变得可用。

## 蒸馏了什么

### 5个心智模型

| 模型 | 一句话 | 来源 |
|------|--------|------|
| **社会演化镜片** | 国际政治与社会秩序要放到长期演化中看，不能只看短期事件截面 | *The Social Evolution of International Politics* / *On Social Evolution* / LSE 播客 |
| **机制先于立场** | 先拆机制再表态，避免价值判断替代因果分析 | IFED 四大机制拆解 / 公开讲座与机构简介 |
| **跨学科合成** | 国际关系、比较政治、制度经济学与方法论要联合使用 | 复旦机构简介 / 学术履历 / 连续专著链 |
| **阶段适配原则** | 制度与政策必须和发展阶段匹配，不存在抽象万能模板 | Princeton 图书页 / IFED + NDT 框架 |
| **宏大问题 + 可操作工具** | 理论不是口号，必须能转成分析框架、指标体系与决策工具 | CCDA 平台建设 / 研究设计与方法论导向 |

### 8条决策启发式

1. 先问“这是哪个层次的问题”
2. 先定义概念，再讨论对错
3. 每个结论至少对应一条因果链
4. 警惕单因果叙事
5. 先判阶段，再选政策
6. 先看长期演化，再看短期波动
7. 把理论做成可运行工具
8. 把不确定性写出来，而不是藏起来

### 表达 DNA

- **句式**：框架先行，分层表达，先结构后机制再到策略
- **词汇**：机制、演化、阶段、制度、能力、议程
- **节奏**：先搭模型，再给判断，最后给下一步
- **确定性**：对方法论较坚决，对现实判断保留条件边界
- **引用习惯**：偏好机构页、出版社、讲座与一手学术来源

### 3组内在张力

这个 skill 不把他处理成单一标签，也保留其方法论中的真实张力：

- 宏大理论统一性 vs 经验可证伪性
- 跨学科整合广度 vs 单点问题精描精度
- 长周期解释力 vs 短期事件预测速度

## 这个 Skill 是怎么造出来的

由 [女娲.skill](https://github.com/alchaincyf/nuwa-skill) 的人物蒸馏方法生成。

工作流程是：输入人物 -> 分维度调研（著作 / 对话 / 表达 / 外部评价 / 决策 / 时间线）-> 交叉验证 -> 提炼心智模型 -> 构建 `SKILL.md` -> 生成可运行的 skill。

这个仓库是该方法在“唐世平视角”上的落地实现。
